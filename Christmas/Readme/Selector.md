# @逻辑选择器

空行输入`@`，IDE会提示所有逻辑选择器。

> 请注意逻辑选择器的层级缩进

提供以下逻辑选择器：

- [多分支选择](#multi-branch-selection)
- [函数调用](#function-call)
- [错误捕获](#error-capture)
- [失败重试](#failed-retry)

**若逻辑过于复杂（使用逻辑选择器也无法实现），则说明这部分逻辑超出了业务逻辑的范畴，应该新建一个模块用代码编写具体的逻辑。**

> “把一头大象放进冰箱需要什么步骤”，“第一步、第二步、第三步的步骤”属于业务逻辑，至于具体“怎么将大象放进冰箱/需要多大的冰箱”等问题不是业务逻辑需要关注的，而是`具体模块`需要解决的

# 多分支选择

多分支选择，相当于大多数编程语言都支持的switch/case。

基本格式如下，拥有相同空格缩进的`@switch`、`@case`，会被视为一组设置：

```
@switch [option]
    ...默认逻辑
@case [value]
    ...分支逻辑
@case ...
```

`[option]`，从passParam中获取值的key，程序会自动根据这个key，获取`passParam`中对应的值，如果希望获取内层嵌套的Json值，可以使用`>>`定位，如`key_1>>key_2`。若值不存在，默认为字符串`"null"`。

`[value]`，分支值，于`[option]`获取到的值进行比较，若匹配成功，则会执行此分支逻辑。

> `[option]`获取到的值会被强制转换为字符串，所以`[value]`不需要考虑值类型（都会被认为是字符串）

例如，对`passParam`中`key`对应的值进行多分支逻辑设置：

```
@switch key
    ...默认逻辑
@case value 1
    ...分支逻辑
@case value 2
    ...分支逻辑
```

# 函数调用

函数调用，调用当前`.xmas`中的自定义函数。

基本格式如下：

```
@call &[function]
```

`[function]`是自定义函数名。

例如，调用`abc`函数。

```
@call &abc
```

调用函数时，数据池会被正常传入，若函数内对数据池进行修改，会被保留。

# 错误捕获

错误捕获，主要用于当正常逻辑失败报错时，运行异常处理逻辑。

基本格式如下，拥有相同空格缩进的`@try`、`@error`，会被视为一组设置：

```
@try
    ...正常逻辑
@error
    ...错误逻辑
```
<a id="failed-retry"></a>

# 失败重试

失败重试，主要针对失败时重新尝试的场景，例如，新生成的8位随机数在数据库中已经存在而报错，需要重新尝试生成新随机数：

基本格式如下：

```
@retry [count] [interval]
	...子逻辑
```

`[count]`是尝试的次数，当子逻辑结束时，如果结果报错，且`retryTime>0`，重新执行一遍逻辑。重新执行逻辑时，会将`passParam`恢复到进入`@retry`之前的状态。

> 无论`[count]`设置为多少，都会至少执行一次，在逻辑执行前，会先减1

`[interval]`是重试间的时间间隔，单位为毫秒，1000=1s，可以设置为0，表示不需要时间间隔。

例如，失败重试10次，每次间隔1秒。

```
@retry 10 1000
	...子逻辑
```
